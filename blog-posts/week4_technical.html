Let's Talk About Maps

Many of the most useful methods in Ruby come from the Enumerable module. One of these methods I'm going to talk about today is the map method. The map method is called on an array and it returns a new array with the results of running whatever code block is passed to it.

For example, if you have an array of numbers, say [5,15,10] and you call [5,15,10].map {|a| a*2} the function would return [10,30,20]. By contrast, if you were to typed [5,15,10].each {|a| a * 2} the function would still return [5,15,10]. Why is this? Because map actually returns a NEW value, which each returns the ORIGINAL array. This makes map extremely useful when you are using the output of a method as an input into another method.
